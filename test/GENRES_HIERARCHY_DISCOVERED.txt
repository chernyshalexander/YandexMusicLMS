ИНТЕРЕСНЫЕ НАХОДКИ
════════════════════════════════════════════════════════════════════════════════════════════════════════

1. PARENTID СОДЕРЖИТ ОБЪЕКТ (не строку!)
   ✓ Тип данных: dict (JSON объект)
   ✓ Структура: {"type": "genre", "tag": "<parent_tag>"}
   ✓ Это позволяет расширять в будущем (например, добавить type: "micro-genre")

2. idForFrom ОТРАЖАЕТ ИЕРАРХИЮ
   ✓ Формат: "genre-<parent>_<child>" или "genre-<single>"
   ✓ Примеры:
     - "genre-allrock_rusrock" → Rock → Russian Rock
     - "genre-electronics_techno" → Electronic → Techno
     - "genre-jazz_tradjazz" → Jazz → Traditional Jazz
   ✓ Это может быть использовано для парсинга иерархии!

3. МИКРОЖАНРЫ НЕ ИМЕЮТ PARENTID
   ✗ Все 367 микрожанров имеют parentId = null/empty
   ✗ Это означает, что микрожанры НЕ связаны явно с жанрами
   ✗ Однако, они могут быть связаны неявно через анализ названий

4. СТРУКТУРА ЖАНРОВ
   ✓ 158 жанров всего
   ✓ 21 родительский жанр (у которых есть дочерние)
   ✓ 130+ дочерних жанр (которые ссылаются на родителя)
   ✓ ~10 корневых жанров (у которых нет parentId и нет дочерних)

КАК ИСПОЛЬЗОВАТЬ ЭТУ ИНФОРМАЦИЮ
════════════════════════════════════════════════════════════════════════════════════════════════════════

1. ПОСТРОИТЬ ИЕРАРХИЮ ЖАНРОВ
   ```
   Browse → Yandex Music → Жанры
   ├── Rock (parent)
   │   ├── Hard Rock (child)
   │   ├── Progressive Rock (child)
   │   └── Russian Rock (child)
   ├── Electronic (parent)
   │   ├── House (child)
   │   ├── Techno (child)
   │   └── Dubstep (child)
   ```

2. РАЗДЕЛИТЬ МЕНЮ
   ```perl
   # Получить только родительские жанры (те, у которых есть children)
   my @parent_genres = grep { has_children($_) } @genres;

   # Для каждого parent получить children через parentId
   my @children = grep { $_->{parentId}->{tag} eq 'rock' } @genres;
   ```

3. ИСПОЛЬЗОВАТЬ idForFrom ДЛЯ ВЕРИФИКАЦИИ
   ```
   genre-rock_hardrock → Rock > Hard Rock (проверяем иерархию)
   ```

❌ ПРЯМЫХ СВЯЗЕЙ НЕТ в API

Возможные подходы:
1. По названиям (например, "Rock" в названии микрожанра)
   ✓ "Hard Rock" → Rock
   ✓ "Progressive Rock" → Rock
   ✓ "Garage Rock" → Rock

2. По иерархии genre (если микрожанр совпадает с дочерним жанром)
   ✓ "hardrock" микрожанр может быть связан с "hardrock" жанром

3. Через mix-by-micro-genre (косвенно через миксы)
   ? mix-by-micro-genre может содержать информацию о связях
